# API and Routing Layer Workflow

This document details the workflow of the FastAPI entry point and the routing controllers.

## 1. `backend/main.py`
The `main.py` file serves as the core entry point for the backend application. Its primary responsibilities include:

- **Application Initialization:** Bootstrapping the FastAPI application instance.
- **Lifespan Management:** Managing the asynchronous lifespan context of the application. During startup, it instantiates the `DETRSAMPipeline`, loading the heavyweight deep learning weights (RT-DETR and SAM) into GPU memory (VRAM) only once. This prevents the overhead of loading models on every request.
- **Middleware Configuration:** Implementing Cross-Origin Resource Sharing (CORS) middleware to allow communication from the frontend client.
- **Health Monitoring:** Exposing an `/api/health` endpoint that returns system telemetry, including CPU, RAM, and GPU utilization, as well as model initialization status.

## 2. `backend/routers/analyze.py`
This module defines the RESTful endpoints responsible for triggering the machine learning pipelines.

### 2.1. `POST /analyze`
This endpoint handles the fully automated end-to-end inference workflow.
1. **Request Parsing:** Accepts the uploaded file and various form parameters (confidence thresholds, windowing parameters, compute device).
2. **Temporary Storage:** Saves the file to a secure, uniquely identified temporary directory (`temp_inputs/`) using `uuid4` to prevent race conditions during concurrent requests.
3. **Data Preprocessing:** Delegates the file to `medical_io.py` to extract a 2D slice if a 3D volume is provided.
4. **Pipeline Execution:** Invokes `pipeline.run_on_image()`, passing the 2D slice and threshold configurations.
5. **Base64 Serialization:** Reads the output files (masks, overlays) generated by the pipeline and converts them into Base64 encoded strings using the `file_to_base64_url` helper function.
6. **Response Formulation:** Constructs a detailed JSON response containing the encoded images and associated metadata, and cleans up the temporary files.

### 2.2. `POST /analyze/point`
This endpoint handles the interactive segmentation workflow.
1. **Prompt Parsing:** Accepts the uploaded image alongside JSON-encoded arrays of point coordinates and labels (foreground/background).
2. **Pipeline Execution:** Invokes `pipeline.run_interactive_prompt()`, which bypasses the detection model and queries SAM directly using the user-provided points.
3. **Response Formulation:** Returns the resulting masks and overlay visualizations to the client.

### 2.3. `POST /windowing`
A lightweight endpoint used for real-time windowing adjustments.
1. **Processing:** Reads the medical volume, applies the requested `window_center` and `window_width`, and extracts the specific `slice_index`.
2. **Response:** Returns only the Base64 representation of the adjusted 2D slice, enabling real-time previewing on the frontend without running the heavy ML models.
